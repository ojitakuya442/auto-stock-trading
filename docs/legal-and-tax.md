# 法律・税務 詳細調査レポート

**対象**: 個人が自己資金（10万円）で自分の口座から自動売買BOTを動かす想定 / 2026年4月時点

> [!IMPORTANT]
> 本ドキュメントは公開資料に基づく一般的整理であり、個別事案については弁護士・税理士に相談すること。
> 判断に迷うグレーゾーンに踏み込む前に必ず専門家確認。

---

## TL;DR

| 観点 | 結論 |
|---|---|
| **登録義務** | 自己資金・自己判断のBOTなら**金商法上の登録不要**（他人に助言/販売/運用代行するとアウト）|
| **禁止行為** | 見せ玉・仮装売買・相場操縦・引け値関与・インサイダー取引 — **個人BOTでも課徴金事例あり** |
| **HFT規制** | 個人の自宅PC・通常APIなら**対象外** |
| **所得区分** | 上場株譲渡益は**申告分離 20.315%固定**。BOT運用が事業所得認定された判例はほぼ無し（雑所得扱い）|
| **推奨口座** | **特定口座（源泉徴収あり）**。確定申告原則不要、損失繰越したい年だけ申告 |
| **副業禁止規定** | 株式投資は通説で**副業に該当しない**。バレ対策は住民税の普通徴収切替 |
| **NISA × BOT** | 相性悪い（損益通算不可・回転売買で枠消費）。10万円規模ならまず特定口座 |
| **経費計上** | 上場株式譲渡所得では**取得費・手数料のみ**。PC/サーバー代等は基本不可 |

## A. 法律・規制面

### A-1. 金融商品取引法（金商法）— 登録義務

#### A-1-1. 自己資金・自己口座の自動売買 → 登録不要

金商法29条で登録義務が課されるのは**「金融商品取引業を業として行う者」**。28条が定める業の類型は、いずれも**「他人のため」**に行うことを前提とする。

| 行為 | 登録要否 |
|---|---|
| 自分の資金を自作BOTで自動売買 | **不要** |
| BOTのコードをGitHubで無償公開 | 不要（具体的な売買助言を伴わない限り） |
| 売買シグナルを有償配信 | **投資助言・代理業 登録必要** |
| 他人の資金を預かって運用 | **投資運用業 登録必要**（最低資本金5,000万円） |
| BOTを販売（具体的な銘柄・売買タイミングを提示） | **投資助言・代理業 登録必要** |

> 単に値動き予想を配信するだけでは投資助言業に該当しないが、**特定の金融商品の売買ポイント等の投資判断を具体的にアドバイスした場合**には規制対象（牛島総合法律事務所）

**根拠条文・出典**:
- [金融商品取引法 第28条・第29条](https://laws.e-gov.go.jp/document?lawid=323AC0000000025_20240201_505AC0000000079)
- [金融庁「投資運用業等 登録手続ガイドブック」](https://www.fsa.go.jp/policy/marketentry/guidebook/01.pdf)
- [牛島総合法律事務所「投資助言業登録が必要となるか否かの判断と行政対応の留意点」(2025)](https://www.ushijima-law.gr.jp/topics/20250421investment_advisory/)
- [金融庁「監督指針 VII（投資助言・代理業）」](https://www.fsa.go.jp/common/law/guide/kinyushohin/07.html)

### A-2. 不公正取引（自動売買で起こりがち）

#### A-2-1. 相場操縦（金商法159条）

金商法159条が禁止する行為:
- **仮装売買**(159条1項): 同一人が売り買い両方を発注し取引が活発と誤認させる
- **馴合売買**(159条1項): 通謀して同時刻・同価格で売買
- **変動操作**(159条2項1号): 「他人を有価証券売買に誘引する目的」で価格を変動させる
- **安定操作**(159条3項): 価格を釘付け・固定する操作

**BOTで特にリスク高い**: 同一口座から同一銘柄に短時間で売買両方を高頻度で出すアルゴリズムは、意図せず**仮装売買と評価されうる**。

#### A-2-2. 見せ玉（みせぎょく / Spoofing）→ **個人BOTでの実例あり**

「**約定意思のない注文発注 → 板を厚く見せて他者誘引 → キャンセル**」の手法。**金商法159条2項1号の変動操作に該当**し、課徴金または刑事罰。

##### 実際の処分事例

| 事例 | 概要 | 処分 |
|---|---|---|
| 個人デイトレーダー(2008年頃) | 1ヶ月、上値売り板/下値買い板を厚く見せて誘引→反対売買→キャンセル反復 | **課徴金 159万円** |
| 引け値関与(2024年勧告) | 個人がオカムラ食品工業株ほか5銘柄、信用取引で「引成」注文を有利に約定させるため**特殊見せ玉**を反復 | 課徴金勧告 |
| 海外45社(2015年) | 私設取引会社による相場操縦 | 課徴金勧告 |
| 野村證券(2024年9月) | 国債先物市場での見せ玉 | 課徴金勧告 |

##### 実務的なNGライン（明確閾値はないが目安）

1. **「約定させる意思の有無」**が判定基準
2. キャンセル率 70-90%超は要注意
3. 反対側に約定意図のある注文があるとアウトになりやすい
4. **「板情報を見て、約定回避のために自動キャンセルする」アルゴリズムは極めて危険**

#### A-2-3. 終値関与・引け値操作

- 板寄せ方式で価格決定する場の直前(14:55-15:00)に価格に影響する注文を出すと「終値関与」と見なされやすい
- **月末・四半期末**の引けは特に監視対象（投信評価額・指数算出に影響するため）

→ **実装上の防御**: BOTで14:55以降の発注は明示的に止めるロジックを入れる

#### A-2-4. 高速取引行為（HFT）規制

2018年4月施行の金商法改正で「高速取引行為者」の登録制が導入(金商法29条の2の4等)。

- 法令上は個人も対象になりうる
- ただし「高速取引行為」の定義は**取引所コロケーション設備等を利用するアルゴリズム取引**で、**通常の個人BOT（自宅PC・通常API経由）は該当しない**
- → **個人投資家10万円・通常APIなら実質無関係**

#### A-2-5. インサイダー取引（金商法166条・167条）

- 上場会社の「会社関係者」が「重要事実」を知って公表前に売買すると刑事罰
- **自分の勤務先株を扱うBOTは作らない**のが鉄則
- BOTの入力データに未公表情報が混じらないよう情報源を管理

## B. 税務面

### B-1. 所得区分と税率

#### 上場株式譲渡益: 申告分離課税 **20.315%**
- 所得税 15% + 復興特別所得税 0.315% + 住民税 5%

#### 配当所得 — 3つの選択肢

| 方式 | 税率 | 損益通算 | 配当控除 |
|---|---|---|---|
| 総合課税 | 累進(15-55%) | 不可 | 適用可 |
| **申告分離課税** | 20.315% | **譲渡損と通算可** | 不可 |
| 申告不要 | 20.315%(源泉のみ) | 不可 | 不可 |

→ **BOTで損失が出た年は申告分離で配当と通算**する設計が定石

#### BOT運用は事業所得になりうるか → **ほぼ不可能**

国税不服審判所の確立した基準:
> 単に営利性・有償性・継続性・反復性のみならず、**事業としての客観性**（人的・物的設備、資金調達方法、精神的・肉体的労力の程度、職歴、社会的地位、安定収益の可能性）が問題

**FX・株のデイトレで「事業所得」と主張した訴訟はほぼ全敗**。

ただし、上場株式譲渡所得は**そもそも申告分離課税で一律20.315%**（措置法37条の10）のため、事業所得認定の議論は実質「経費計上の幅」にしか影響しない。

**出典**:
- [国税不服審判所「有価証券の継続的売買による所得」](https://www.kfs.go.jp/service/MP/02/0208030000.html)
- [公表裁決事例(令和3年11月19日)](https://www.kfs.go.jp/service/JP/125/02/index.html)
- [措置法37の10 関係](https://www.nta.go.jp/law/joho-zeikaishaku/shotoku/joto-sanrin/040715/02.htm)

### B-2. 口座種別

| 口座 | 確定申告 | 損益通算 | 推奨度 |
|---|---|---|---|
| **特定口座（源泉徴収あり）** | 原則不要 | 申告すれば可 | ★ **BOT運用者の標準** |
| 特定口座（源泉徴収なし） | 必要 | 可 | 自分で計算したい人向け |
| 一般口座 | 必要（自分で集計） | 可 | 非推奨 |
| NISA口座 | 不要（非課税） | **不可** | 別枠で長期投資に |

**重要**: 源泉徴収あり特定口座でも、**損失繰越控除を使うなら確定申告必要**。

[国税庁 No.1476 特定口座制度](https://www.nta.go.jp/taxes/shiraberu/taxanswer/shotoku/1476.htm)

---

### B-3. 確定申告

#### 給与所得者の20万円ルール（所得税法121条）

給与1か所・年末調整済みで、**給与・退職以外の所得合計が20万円以下**なら所得税の申告不要。

**重要な注意**:
- 特定口座（源泉徴収あり）の譲渡益・配当は「申告不要」を選択でき、20万円ルールの計算に**含めない**
- **住民税には20万円ルールはない**（地方税法上）。20万円以下でも住民税申告は必要 → 源泉徴収ありなら住民税も完結

#### 損益通算と繰越控除

- 同一年内: 上場株式譲渡損失 ⇔ 譲渡益 / 申告分離配当 と通算可
- 通算しきれない損失: **翌年以降3年間繰越可**（措置法37条の12の2）
- 繰越中は**毎年連続して申告必要**（1年でも飛ばすと打ち切り）

**出典**:
- [国税庁 No.1900 給与所得者で確定申告が必要な人](https://www.nta.go.jp/taxes/shiraberu/taxanswer/shotoku/1900.htm)
- [上場株式等に係る譲渡損失の損益通算及び繰越控除](https://www.keisan.nta.go.jp/r4yokuaru/cat2/cat21/cat219/kabushikijototokurei/sonekitsusan.html)

### B-4. 経費計上

上場株式譲渡所得は申告分離・譲渡所得のため、計上できる経費は**取得費・譲渡費用（手数料）のみ**。

- BOT開発のPC、サーバー代、API利用料、書籍代は**譲渡所得の経費としては基本的に認められない**
- 例外: 信用取引の金利・貸株料は譲渡費用に含められる
- 雑所得部分（FX・暗号資産）であれば必要経費として広く計上可能

→ **これがBOT運用者にとって最大の論点**。「PC代を経費にできる」というネット情報は雑所得（FX/暗号資産）の話であって、上場株式譲渡所得には当てはまらない。

### B-5. 仮想通貨と株式の違い

| 項目 | 上場株式 | 暗号資産 |
|---|---|---|
| 所得区分 | 譲渡所得（申告分離） | **雑所得（総合課税）** |
| 税率 | 20.315%固定 | 累進 5-55%(住民税込) |
| 損益通算 | 株式間・配当と通算可 | **株式と通算不可** |
| 繰越控除 | 3年間可 | **不可** |
| 経費計上 | 取得費・手数料のみ | 広く必要経費可 |

**両方やる場合の鉄則**: 帳簿を完全に分離。仮想通貨BOTで損失が出ても株BOTの利益から差し引けない。

[国税庁「暗号資産等に関する税務上の取扱い(令和7年12月)」](https://www.nta.go.jp/publication/pamph/shotoku/kakuteishinkokukankei/kasoutuka/index.htm)

### B-6. 海外証券（Alpaca / IBKR / 米国株）

#### 米国株の課税

- **譲渡益**: 国内のみで申告分離課税 20.315%（米国では非課税）
- **配当**: 米国で源泉10%（租税条約） → 残額に日本で20.315% → **二重課税**
- **解消**: **外国税額控除**を確定申告で申請

#### 為替差損益

- 株取引に伴う為替差損益: 譲渡所得の計算に含めて処理（円換算）
- 純粋なFX: 雑所得として総合課税

#### 海外証券（Alpaca等、日本に支店なし）

- 特定口座が使えないため**一般口座扱い** → 自力で年間損益計算が必須
- 円換算は約定日のTTM等で行う

---

### B-7. NISA × 自動売買 → **相性悪い**

- NISA口座でAPI発注は技術的に可能（口座種別を指定できれば）
- 制約:
  - 年間投資枠 240万円(成長投資枠)/120万円(つみたて枠)、生涯枠1,800万円
  - **損益通算不可・繰越控除不可・売却損は税務上ないものとみなされる**
- BOT（短期売買）とNISAは相性が悪い: 損切りが税務上活かせない、回転売買で年間枠を消費しやすい
- **10万円規模ならまず特定口座から始めるのが妥当**

## C. 実務的なグレーゾーン

### C-1. 副業禁止のサラリーマンが自動売買BOTを動かすのは副業か

**通説**: **株式投資・FXは「副業」に該当しない**（事業ではないため）

例外:
- 銀行・証券会社など金融機関は社内規程で投資自体を制限する場合あり
- インサイダー対象会社の従業員は所属会社の自社株BOTは厳禁
- 業務PC・会社回線を使うのはアウト（就業時間中にBOT稼働自体は問題ない）

**バレ対策（住民税）**:
- 特定口座（源泉徴収あり）を選び確定申告しなければ住民税通知に載らない
- 確定申告するなら住民税を「**自分で納付（普通徴収）**」に切り替える

### C-2. 個人事業主としての開業届

- 株式譲渡所得は事業所得認定されないため、**BOT運用を理由とした開業届は意味が薄い**
- FX・暗号資産で大きな利益を継続的に出している場合に検討余地

### C-3. 法人成り

- 10万円規模では論外
- 一般的に**継続的に年800万円超の運用益**が見込めるレベルで検討
- 法人税実効率 約30%、損失繰越10年、経費の幅広い計上、役員報酬での所得分散が可能

### C-4. 証券会社にBOT利用を申告する義務

- **義務はない**
- API公開しているSBI/楽天/auカブコム/立花は事実上OK
- 大量発注で異常検知された場合、口座一時停止のリスクあり（各社規約による）

## E. 設計書への落とし込み — 推奨スタックとレッドライン

### E-1. 推奨構成（10万円・個人BOT）

| 項目 | 推奨 |
|---|---|
| 口座 | **特定口座（源泉徴収あり）** — SBI/楽天/auカブコム/立花 |
| 取引対象 | 国内上場株式 |
| 資金 | **自己資金のみ** |
| 助言・配信 | しない（=登録不要を維持） |
| キャンセル率 | 低く保つ（見せ玉的発注禁止） |
| 時間制約 | **14:55以降の発注を禁止**（引け関与回避） |
| ログ保管 | 発注理由・約定意図をトレース可能な形で**1年以上保存** |
| 年次対応 | 12月末損益計算 → 損失あれば確定申告で繰越 |
| 住民税 | 申告するなら**普通徴収**を選択（副業バレ対策）|

### E-2. レッドフラグ（絶対やらない）

1. ❌ 約定意思のない注文を出して即キャンセル
2. ❌ 同一銘柄に売り注文・買い注文を同時に並べる（仮装売買リスク）
3. ❌ 引け前5分の意図的な大口発注
4. ❌ 他人にシグナル配信・BOT販売・運用代行
5. ❌ 勤務先関連企業の銘柄を扱う
6. ❌ 雑所得（FX/暗号資産）と株式の損益計算を混ぜる
